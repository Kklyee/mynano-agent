# Agent 升级路线

这份文档用来排优先级，不展开实现细节。

## P0

### 1. 取消 Run

- 当前只有前端中断，没有真正停止运行
- 这是运行控制面的基础能力

### 2. Tool Trace

- 需要给每次工具调用稳定身份
- 方便调试、审计和回放

### 3. Parse Guard

- 需要为工具参数解析补容错
- 避免 `JSON.parse` 直接打断整轮运行

### 4. Tests

- 现在没有自动化测试
- 后续继续改 runtime 风险很高

## P1

- Background 恢复
- Budget 控制
- Tool Policy
- 结构化日志

## P2

- Planner/Executor
- 多子代理并发
- 长期记忆
- 审批节点

## 一句话

先补控制面，再补可观测，再补复杂能力。
