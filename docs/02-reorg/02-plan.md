# Agent 重组计划

## 结论

可以重组，而且应该重组。

但只能做收敛式重组，不能做大爆炸重写。

## 重组目标

- 保留当前主干行为
- 收清职责边界
- 给后续 cancel 和 trace 留出落点

## 最值得先重组的地方

### 1. Session

目标：

- 更像状态机
- 少塞辅助细节

### 2. Tools

目标：

- 分开 tool 契约、定义、执行入口
- 为横切能力预留位置

### 3. Conversation Handler

目标：

- 只做 HTTP/SSE 桥接
- 持久化翻译逻辑逐步外移

### 4. Runtime

目标：

- 保持它只是装配器
- 不接运行时临时状态

## 重组原则

1. 先改边界，不改协议
2. 先抽共性，不追求花哨抽象
3. 一次只收敛一小块
